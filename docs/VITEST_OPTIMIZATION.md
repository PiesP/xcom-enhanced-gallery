# 🚀 Vitest 성능 최적화 가이드

> **로컬 개발 환경에서 빠른 테스트 실행을 위한 최적화 설정**

## 📊 테스트 모드별 성능 비교

| 모드        | 스레드 수           | 커버리지 | 격리 | 용도         |
| ----------- | ------------------- | -------- | ---- | ------------ |
| `turbo`     | CPU의 85% (최대 12) | ❌       | 완화 | 🚀 최고 속도 |
| `default`   | CPU의 60% (최대 6)  | ✅       | 완전 | 🔄 일반 개발 |
| `optimized` | CPU의 70% (최대 8)  | ✅       | 완전 | 📊 품질 검증 |
| `single`    | 1 (forks)           | ✅       | 완전 | 🐛 디버깅    |

## 🎯 추천 사용법

### 일상적인 개발 작업

```bash
# 빠른 핵심 테스트만 (추천)
npm run test:quick

# 터보 모드 - 최고 속도
npm run test:turbo

# 기본 모드 (멀티스레드 개선됨)
npm test
```

### 디버깅 및 문제 해결

```bash
# 단일 스레드로 안정한 디버깅
npm run test:single

# 감시 모드로 개발
npm run test:turbo:watch
```

### 품질 검증

```bash
# 커버리지 포함 최적화 테스트
npm run test:local

# 전체 테스트 스위트
npm run test:optimized
```

## ⚡ 성능 향상 요소

### 1. 멀티스레드 최적화

- **기존**: Windows에서 단일 스레드 강제
- **개선**: CPU 코어 수에 따른 동적 스레드 계산
- **결과**: 2-6배 속도 향상

### 2. Turbo 모드 특징

- 커버리지 비활성화로 오버헤드 제거
- 테스트 격리 완화로 메모리 사용량 감소
- 핵심 테스트만 선별 실행
- 캐싱 활성화

### 3. 타임아웃 최적화

- Turbo: 5초 (빠른 실패)
- 기본: 6초 (안정성)
- CI: 28초 (네트워크 고려)

## 🔧 환경 변수 설정

```bash
# 수동으로 스레드 수 조정
set VITEST_MIN_THREADS=2
set VITEST_MAX_THREADS=8

# 특정 모드 강제
set VITEST_MODE=turbo
```

## 📈 예상 성능 개선

| CPU 코어 | 기존 (단일) | 개선 (멀티) | 성능 향상  |
| -------- | ----------- | ----------- | ---------- |
| 4코어    | 1 스레드    | 2 스레드    | **2배**    |
| 8코어    | 1 스레드    | 4 스레드    | **4배**    |
| 16코어   | 1 스레드    | 6 스레드    | **6배**    |
| 24코어+  | 1 스레드    | 6-12 스레드 | **6-12배** |
